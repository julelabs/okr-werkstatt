Design System

OKRKritikBot 3000

Komponenten, Tokens und Seitenarchitektur. Basis: die Startseite. Ziel: alle Seiten fühlen sich wie ein System an — gleicher Container, gleiche Nav, gleiche Farbsprache.

01 — Farben & Tokens

Vier Farben. Klare Rollen.

Keine neuen Töne einführen. Minze = Belohnung/Aktion. Kirsche = Kritik/Problem. Blau = Struktur. Lavendelgrau = ruhige Fläche.

Kosmisches Blau
#36349D
Nav, Headings, Struktur, Vertrauen
Minze
#4DFFB0
Primär-CTA, Erfolg, Zielzustand, Vorschlag
Kirsche
#DB0045
Kritik-Chips, Unterstreichung, Fehler
Lavendelgrau
#E3E6F2
Seiten-BG, Hero-BG, neutrale Flächen, Buttons neutral
Dunkel
#0D0C2B
Headlines, Fliesstext, CTA-Tinte auf Minze
Weiß
#FFFFFF
Cards, Panels, Formulare, Nav-BG

Wichtig: Minze nie als dominante Seitenfläche. Nur als Akzent: CTA-Button, positiver Chip, Vorschlag-Balken, Erfolg-Badge. Kirsche nur für echte Fehler — nicht dekorativ.

Einheitliche Topbar auf allen Seiten.

Weiß-Glas, kein dunkles Nav mehr. Gleicher Max-Width-Container (1160px) überall. Aktive Seite: Linkfarbe Blau.

03 — Feedback-Chips

Kleine Labels, klare Einordnung.

Chips klassifizieren OKR-Elemente sofort. Zwei Kategorien: Kritik (Kirsche) und positiv (Minze). Neutral für Meta-Labels.

Kritik-Chips — Kirsche
Kein Objective Maßnahme Unmessbar Zu vage Output, kein Outcome Doppelung
Positiv-Chips — Minze
✓ Zielzustand ✓ Messbar ✓ Klarer ✓ Besser so ✓ Outcome
Neutral / Meta
OKR-Set Entwurf Vorschlag Objective Key Result Folgt

Größe: 11.5px, padding 4px 11px. Kirsche-Chips: immer nur für echte OKR-Qualitätsprobleme. Minze-Chips: immer für Verbesserung und Zielzustand. Label kurz und direkt — kein ganzer Satz.

04 — Dokument-Cards & Feedback-Panels

Das annotierte OKR-Snippet.

Cards zeigen Entwurf + Kritik + Vorschlag. Panels kommentieren im Randnotiz-Stil. Weißer Untergrund, leichter Schatten, kein dunkler Hintergrund.

OKR-Set Entwurf
Objective
12 Fachverfahren bis Q3 in die Cloud migrieren.
Kein Objective Lieferung, kein Zielzustand.
OKR-Set Entwurf
Key Result 1
Monatlich 3 Workshops zum Thema Cloud durchführen.
Maßnahme Aktivität, kein Ergebnis.
Vorschlag
Fachverfahren laufen stabil, sicher und für Teams spürbar einfacher in der Cloud.
✓ Zielzustand
Feedback-Panels
Problem
Das ist eine Maßnahme, kein Key Result. Key Results beschreiben den messbaren Zustand am Quartalsende.
Verbesserung
Dieser Vorschlag beschreibt einen beobachtbaren Zielzustand. Prüfe noch, ob er am Quartalsende eindeutig messbar ist.
Hinweis
Ein OKR-Set besteht typischerweise aus 1 Objective und 2–4 Key Results.

Card-Struktur: DC-Meta (Label) → DC-Field (Element-Typ) → DC-Text (ggf. Kirsche-Unterstreichung) → DC-Ann (Chip + Notiz). Vorschlag-Cards: Minze-Balken links statt Chip oben. Panels: linker Farbbalken = semantische Farbe.

06 — Quiz-Karte & Antwortzustände

Helle Karte, konsistent mit dem System.

Weiße Karte auf Lavendelgrau. Vier Antwortzustände: neutral, ausgewählt, richtig, falsch.

Neutral: Lavendelgrau BG. Ausgewählt: Blau-10 BG, Blau-Border. Richtig: Minze-Tint, Grün-Text. Falsch: Kirsche-Tint, Kirsche-Text. Die Quiz-Seite läuft auf genau diesem hellen System.

07 — CTA-Block & Buttons

Drei Button-Varianten, ein Primär-CTA.

Minze ist immer die Primäraktion. Outline-Blau für die Alternative. Voll-Blau für externe Links.

08 — Empty State, Loading & Eingabe

Bot-Workspace: vor und während.

Empty State bevor der Nutzer etwas eingegeben hat. Skeleton-Loading während das Feedback generiert wird.

Empty State

Noch kein OKR-Set

Füge dein Objective und Key Results ein — du bekommst sofort Feedback.

OKR einfügen

Loading State

Eingabefeld — Bot-Workspace

Empty: Zentriert, Icon in Lavendel-Box, klarer CTA. Loading: Skeleton mit gleicher Struktur wie späteres Feedback — kein Spinner, kein dunkler Overlay. Input: Weiß, Blau-Focus-Ring.

09 — Seitenarchitektur

So sieht das System auf jeder Seite aus.

Gleicher Container, gleiche Nav, unterschiedliche Inhaltsstruktur. Alle Seiten teilen denselben hellen Hintergrund und dieselbe Farbsprache.
